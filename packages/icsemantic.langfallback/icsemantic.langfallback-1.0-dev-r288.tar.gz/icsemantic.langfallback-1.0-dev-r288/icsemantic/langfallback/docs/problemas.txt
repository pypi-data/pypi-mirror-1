En general cuando existe algun problema debemos generar un test en test.bugs
un doctest en lo posible, en donde se explique que pasa y como se pudo
solucionar.

- Problemas con GenericSetup y python products en Plone2.5

	El problema empezo cuando cambie  el configure.zcml de icsemantic.langfallback
	para que registre el profile de GS. El buildout de 2.5 corria pero despues
	se rompia la instancia.
	El problema es que parece que no se cargaba el meta.zml de GenericSetup
	antes que se cargue el del prod. asi que no sabia nada sobre la directiva
	esa...
	Hay dos posibles soluciones,
		- por producto, incluir la siguiente linea en el configure.zcml

		<include package="Products.GenericSetup" file="meta.zcml" />

		- en forma global, modificar el buildout para generar un zcml slug
		  para GenericSetup

		# If you want to register ZCML slugs for any packages, list them here.
		# e.g. zcml = my.package my.other.package
		zcml =
        	Products.GenericSetup-meta

- Problemas con el testbrowser

	En alguna parte se setea mal el HTTP_REFERER o no se pasan los parametros
	necesarios. Si un metodo en el sitio depende de esa variable no puede
	volver a la pagina de origen y se rompen los tests.
	Esta documentado por Martin Aspeli, pero parece que nadie le dio bola...
	https://bugs.launchpad.net/zope3/+bug/98437
	Para solucionarlo cree un monkeypatch y ahora no tenemos mas este
	problema.
